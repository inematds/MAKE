1

Introducao ao Airtable

O que e:

Airtable e um banco de dados visual que combina a simplicidade de planilha com poder de banco relacional. Interface amigavel, API poderosa, perfeito para automacoes.

Por que aprender:

Superior ao Google Sheets para dados estruturados. Permite relacionamentos entre tabelas, views customizadas, e integracao nativa com Make.

Conceitos-chave:

Base, Table, Record, Field, View, Linked records, Formulas, Automations.

2

Google Sheets como banco de dados

O que e:

Usar planilhas como repositorio de dados para automacoes. Simples de configurar, gratis, familiar para 99% dos usuarios.

Por que aprender:

Solucao mais acessivel para armazenamento de dados. Ideal para prototipagem rapida e operacoes de pequeno/medio porte.

Conceitos-chave:

Header row, Data range, CRUD operations, Sheet ID, Range notation (A1:Z100).

Sheets vs Airtable

Google Sheets:

Gratis, simples, otimo para comecar

Airtable:

Relacionamentos, views, melhor para escala

3

Lendo e gravando dados

O que e:

Operacoes basicas de banco de dados: buscar registros existentes (Read) e adicionar novos (Create). Mapear campos corretamente entre sistemas e fundamental.

Por que aprender:

Toda automacao util manipula dados de alguma forma. Sem dominar leitura e escrita, automacoes sao apenas notificacoes simples.

Conceitos-chave:

Read operation, Write operation, Field mapping, Data types, Validation, Upsert.

Mapeamento de Campos

Webhook         →    Planilha
────────────────────────────────
body.nome       →    Coluna A (Nome)
body.email      →    Coluna B (Email)
body.telefone   →    Coluna C (Telefone)
4

Filtros e buscas

O que e:

Encontrar registros especificos usando criterios de busca. Filtrar dados por campo, valor, data, ou combinacoes complexas com AND/OR.

Por que aprender:

Dados uteis precisam ser encontrados. Filtros eficientes evitam processar dados desnecessarios e economizam operacoes.

Conceitos-chave:

Filter conditions, Search query, Match criteria, AND/OR logic, Wildcards, Contains.

Exemplos de Filtro

  • Status = "Ativo" - Busca exata
  • Email contains "@gmail" - Busca parcial
  • Data > 2024-01-01 - Comparacao de data
  • Valor >= 1000 AND Status = "Pago" - Multiplas condicoes
5

Iteradores: processando listas

O que e:

Modulo especial que pega uma lista/array de itens e processa cada um individualmente. Essencial para operacoes em lote como enviar email para lista de contatos.

Por que aprender:

Maioria dos dados vem em listas (leads, produtos, emails). Sem iterador, impossivel processar multiplos registros de forma eficiente.

Conceitos-chave:

Iterator module, Array, Bundle, Loop, Index, Total count, Flow control.

6

Agregadores: consolidando resultados

O que e:

Modulo que combina multiplos bundles em um so. Util para criar resumos, totalizadores, ou consolidar dados de multiplas fontes em uma unica saida.

Por que aprender:

Complemento essencial do iterador. Permite criar relatorios, somas, medias, listas consolidadas a partir de dados processados individualmente.

Conceitos-chave:

Array aggregator, Text aggregator, Numeric aggregator, Group by, Summarize.

Tipos de Agregadores

Array

Junta em lista

Text

Concatena textos

Numeric

Soma, media, etc

Caso de Uso

Iterar sobre vendas do mes → Agregar com soma → Resultado: total de vendas em um unico valor.
